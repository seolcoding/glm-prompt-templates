# 코드 리뷰 템플릿

## 일반 코드 리뷰

```
당신은 시니어 소프트웨어 엔지니어입니다.

다음 코드를 리뷰해주세요:

```[언어]
[코드 붙여넣기]
```

**리뷰 관점**:
1. 기능적 정확성 - 의도한 대로 동작하는가?
2. 성능 - 최적화할 부분이 있는가?
3. 보안 - 취약점이 있는가?
4. 가독성 - 이해하기 쉬운가?
5. 유지보수성 - 확장/수정이 쉬운가?
6. 테스트 - 테스트가 필요한 부분은?
7. 디자인 패턴 - 더 나은 패턴이 있는가?

**출력 형식**:
## 🔴 Critical (반드시 수정)
- [사항]

## 🟡 Suggestion (권장)
- [사항]

## 🟢 Good (잘한 점)
- [사항]

## 💡 개선된 코드 제안
```[언어]
[개선된 코드]
```
```

## PR 리뷰 코멘트

```
당신은 건설적인 코드 리뷰어입니다.

다음 변경사항에 대한 리뷰 코멘트를 작성해주세요:

**변경 내용**: [무엇을 변경했는지]
**변경 이유**: [왜 변경했는지]

**요구사항**:
- 구체적이고 실행 가능한 피드백
- "왜" 설명 포함
- 대안 제시
- 긍정적인 피드백도 포함
- 톤: 건설적이고 협력적인

**형식**:
`[파일명]:[라인]` 기준으로:
- 문제점
- 이유
- 제안
```

## 리팩토링 제안

```
당신은 클린 코드 전문가입니다.

다음 코드를 리팩토링 제안해주세요:

```[언어]
[코드]
```

**목표**:
- [ ] 단일 책임 원칙
- [ ] 중복 제거 (DRY)
- [ ] 가독성 향상
- [ ] 성능 최적화
- [ ] 테스트 용이성

**출력**:
1. 현재 코드의 문제점 분석
2. 리팩토링 단계별 계획
3. 리팩토링된 코드
4. 변경 전/후 비교
5. 주의사항
```
